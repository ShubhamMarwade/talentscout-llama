## 🌟 Overview

TalentScout automates the most time-consuming part of technical hiring — the initial screening. Candidates fill in their profile and tech stack, then receive a fully personalized set of interview questions generated by a local LLM. Every answer is scored individually by AI across three dimensions: **technical depth**, **practical knowledge**, and **communication clarity**.

Recruiters log in to a separate dashboard to review reports, filter candidates by verdict, and export data — all without sending a single byte to a third-party API.

---

## ✨ Features

### For Candidates
- **Personalized Question Generation** — Questions are tailored to the candidate's exact tech stack, experience level, and desired role
- **Real-time Word Count Feedback** — Live indicator encourages detailed, high-quality answers
- **Skip & Finish Early** — Flexible flow; candidates can skip questions or stop after at least 5 answers
- **Completion Ring** — Visual summary of answered vs. skipped questions on the result screen
- **Secure & Local** — No data ever leaves your infrastructure

### For Recruiters
- **Protected Portal** — Login-gated dashboard separate from the candidate flow
- **Dashboard Overview** — Stats grid showing total candidates, hire/maybe/reject counts, and average score at a glance
- **Search & Filter** — Filter candidates by recommendation verdict or search by name/email
- **Detailed Reports** — Per-question scores, evaluator notes, verdict banner, strengths/weaknesses analysis
- **JSON Export** — Download any candidate's full report as a structured JSON file
- **Delete Candidates** — Remove records with a confirmation guard

### AI Evaluation Engine
- Per-question scoring (1–10) with a one-sentence evaluator note for each answer
- Aggregated metrics: Technical Depth, Practical Knowledge, Communication
- Three-tier recommendation system: **Hire** / **Maybe** / **Reject**
- Graceful fallback scoring when the LLM response cannot be parsed

## 📁 Project Structure

```
talentscout/
│
├── app.py              # Main application — routing, session state, phase logic
├── components.py       # HTML component library (returns strings for st.markdown)
├── styles.py           # PREMIUM_CSS — complete dark design system
├── llm.py              # Ollama API wrapper, streaming, JSON extraction
├── prompts.py          # SYSTEM_PROMPT and TECH_Q_PROMPT templates
├── data_store.py       # Candidate persistence: save, load, delete, stats
│
└── README.md
```

### Key Modules

**`app.py`** — Phase-based state machine with four candidate phases (`PROFILE → TECH_QA → EVALUATING → SHOW_EVALUATION`) and a separate recruiter flow (`dashboard ↔ report`). Contains the full evaluation engine (`_run_evaluation`, `_score_one_answer`).

**`components.py`** — Pure functions that return HTML strings. Every component exactly matches the HTML prototype design: `question_card()`, `score_card()`, `recommendation_banner()`, `completion_ring()`, `candidate_row()`, `stats_grid()`, and more.

**`styles.py`** — A single `PREMIUM_CSS` string injected at startup. Defines all CSS custom properties, component classes, animation keyframes, and Streamlit widget overrides.

**`llm.py`** — Handles communication with the Ollama API including streaming responses, safe JSON extraction, and retry logic.

**`data_store.py`** — Manages all local persistence. Generates deterministic candidate IDs, saves evaluation reports, and provides aggregated statistics for the dashboard.

## 🚀 Getting Started

### Prerequisites

- Python **3.11** or higher
- [Ollama](https://ollama.com) installed and running locally
- A compatible LLM pulled in Ollama (e.g. `llama3`, `mistral`, `gemma`)

```bash
# Install Ollama (macOS / Linux)
curl -fsSL https://ollama.com/install.sh | sh

# Pull a model
ollama pull llama3
```

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-username/talentscout.git
cd talentscout
```

**2. Create and activate a virtual environment**

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

A minimal `requirements.txt`:

```
streamlit>=1.32.0
requests>=2.31.0
```

**4. Configure your model**

Open `llm.py` and set the model name and Ollama host to match your setup:

```python
OLLAMA_MODEL = "llama3"          # or mistral, gemma, etc.
OLLAMA_HOST  = "http://localhost:11434"
```

### Running the App

```bash
streamlit run app.py
```

The app will open at **http://localhost:8501**

### Evaluation Scoring

Each answer is scored 1–10 by the LLM. Aggregate scores are computed as:

| Metric | Calculation |
|---|---|
| **Technical Depth** | Weighted average (35% early questions, 65% later questions) |
| **Practical Knowledge** | Average of non-skipped answers |
| **Communication** | Score + bonus for answer length, capped at 10 |
| **Overall Score** | Simple average of all individual scores |

| Overall Score | Recommendation |
|---|---|
| ≥ 7 | ✅ **Hire** |
| 5 – 6 | ⚠️ **Maybe** |
| ≤ 4 | ✗ **Reject** |

## ⚙️ Configuration

All key configuration lives in two files:

**`app.py`**
```python
RECRUITER_EMAIL    = "recruiter@demo.com"
RECRUITER_PASSWORD = "demopass123"
```

**`llm.py`**
```python
OLLAMA_MODEL = "llama3"
OLLAMA_HOST  = "http://localhost:11434"
```

**Question generation limits** — By default, questions are capped at 15. To change this, find the slice in `app.py`:

```python
flat = flatten_questions(questions_list)[:15]  # change 15 to your desired cap
```

**Scoring retry attempts** — The LLM question generator retries up to 3 times on failure:

```python
if st.session_state._generation_attempts >= 3:  # change threshold here
```
